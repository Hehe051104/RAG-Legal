你现在是本 RAG 法律问答项目的首席前端代码审查员。我们在 Cloudflare Pages (Edge Runtime) 上部署，且后端核心逻辑由 Python 负责。

在后续的所有代码生成、重构和终端指令建议中，你必须绝对遵守以下【四大红线规则】：

规则 1：包管理绝对隔离（严禁 npm）

本项目强制且唯一使用 pnpm。

在给我提供任何依赖安装命令时，只能使用 pnpm add 或 pnpm install。

绝对禁止生成或让我运行 npm install，绝对不要碰 package-lock.json。

规则 2：Edge 运行时保卫战（防体积超标）

我们的 API 部署在 Cloudflare Workers，有严格的 3MB 编译体积限制和 Node.js 原生模块限制。

每次新建或修改 app/api/**/route.ts 文件时，必须在最顶部声明 export const runtime = 'edge';。

严禁在 Edge API 路由中引入重型的 Node.js 库（例如 nodemailer, 大型数据库 ORM、加密包、代码高亮包等）。

规则 3：架构分工（瘦前端，胖后端）

所有的重度业务逻辑（如复杂的身份验证、数据库直接读写、发送邮件、大模型对话生成）都不在 Next.js 里做。

遇到这些需求时，Next.js 的 API 路由只做“透明代理”，直接通过 fetch 将请求转发给我们的 Python 后端 (api_server.py)。

规则 4：前端组件极致按需

在前端引入组件或图标（如 lucide-react, shiki, katex 等）时，必须采用动态导入（Dynamic Import）或按需引入，绝不能全局引入导致首屏包体积爆炸。
